## なぜ3つを区別するのか

Gitの学習では、GitとGitHubが同じものに見えやすくなります。しかし、どこに履歴があるか分からないと「CommitしたのにGitHubで見えない」と迷います。まず役割を分けましょう。

## 3つの登場人物

| 名前           | 役割                                 | 動く場所       |
| -------------- | ------------------------------------ | -------------- |
| Git            | ファイルの変更履歴を管理する仕組み   | 主に自分の端末 |
| GitHub         | リポジトリを置き、共有できるサービス | Web上          |
| GitHub Desktop | Gitを画面から操作するアプリ          | 自分の端末     |

GitHub DesktopはGitの状態を見やすくし、ボタン操作をGitへ伝えます。GitそのものやGitHubの代わりではありません。

<Callout type="tip" title="ローカルとリモート">
  ローカルは自分の端末側、リモートはネットワーク上の相手側です。このコースのリモートはGitHub上のリポジトリです。
</Callout>

## 操作前の状態

まだ練習用リポジトリはありません。GitHub Desktopをインストールしてサインインしていても、学習用の履歴は作られていません。

## 手順：自分の言葉で言い換える

次の文を声に出すか、メモしてください。

- Gitは、端末内を中心に変更履歴を管理する仕組みです。
- GitHubは、リモートリポジトリを置いて共有するWebサービスです。
- GitHub Desktopは、Gitを画面から操作するためのアプリです。

## 操作後の状態

ファイルや履歴はまだ変わりません。ただし、次のレッスンから画面を見たときに「いまGitHubを操作している」「いま端末内のGitを操作している」と区別できます。

## 内部で起きたこと

このレッスンも概念の準備です。GitHub DesktopでCommitするとローカルリポジトリが変わり、Pushするとリモートリポジトリが変わる、という違いが今後の軸になります。

<OptionalDetails summary="リポジトリとは">
  リポジトリは、プロジェクトのファイルとGitの履歴を管理する単位です。作業フォルダの中には通常、Gitが使う隠し領域もあります。GitHub上のリポジトリには作業中の未Commit変更はありません。
</OptionalDetails>

## よくある失敗

GitHub Desktopにファイルが見えるので「GitHubにアップロード済み」と思うことがあります。画面に見えている変更がまだCommit前なら作業フォルダだけ、Commit後でもPush前ならローカルだけにあります。

## 対応するGitコマンド

役割を確認するだけなので、まだコマンドは不要です。端末内にGitが入っているかを確認するときは、任意のフォルダで `git --version` を実行できます。このコマンドはファイルや履歴を変更しません。

```bash
git --version
```

## なぜCloneするのか

GitHub上のリモートリポジトリだけでは、端末のエディターでファイルを編集できません。Cloneは、リモートのファイルと履歴を端末へ複製し、今後のPushやPullに使う接続情報も設定する操作です。

## 操作前の状態

GitHub上にはREADMEと履歴がありますが、端末内にはありません。

## 手順：GitHub DesktopでCloneする

1. GitHub Desktopを開きます。
2. **File → Clone repository…** を選びます。
3. **GitHub.com** タブで `git-guidebook-practice` を選びます。
4. **Local path** を確認します。個人名が見える場所をスクリーンショットへ載せないでください。
5. **Clone** を選び、処理が終わるまで待ちます。
6. **Repository → Show in Explorer**（macOSではShow in Finder）で保存場所を開きます。

## 内部で起きたこと

GitはリモートのCommit履歴とファイルを取得し、端末内にローカルリポジトリを作りました。さらに、取得元のリモートを通常 `origin` という名前で登録しました。

<OptionalDetails summary="ダウンロードZIPとの違い">
  ZIPのダウンロードは現在のファイルをコピーしますが、通常はGitの履歴やリモートとの接続情報を含みません。Cloneは履歴を含むリポジトリとして取得するため、その後Commit、Push、Pullできます。
</OptionalDetails>

## よくある失敗

すでに同名フォルダがあり、中にファイルがあるとCloneできないことがあります。既存フォルダを消さず、別の安全な保存先を選んでください。

## 対応するGitコマンド

親フォルダで実行します。URLは自分の教材用リポジトリのものへ置き換えます。成功すると作業フォルダとローカルリポジトリが作られます。

```bash
git clone https://github.com/USER/git-guidebook-practice.git
```

## なぜBranchを作るのか

mainは、プロジェクトの基準となる履歴です。新しい変更を別のBranchで進めると、完成・確認するまでmainを動かさずに済みます。Branchはファイルを入れる別フォルダではなく、履歴の進む先を分ける仕組みです。

## 操作前の状態

ローカルとGitHubのmainは同じCommitを指し、未Commit変更はありません。

## 手順：Branchを作成してPublishする

1. GitHub Desktop上部の **Current Branch** を開きます。
2. **New Branch** を選びます。
3. 名前に `add-learning-goal` と入力します。
4. **Create Branch** を選びます。
5. 上部表示が `Current Branch: add-learning-goal` になったことを確認します。
6. **Publish branch** を選び、GitHub上にも同名Branchを作ります。

## 内部で起きたこと

ローカルに新しいBranch名が作られ、現在地がそのBranchへ切り替わりました。Publish branchで同じBranch名がリモートにも作られ、今後Pushする対応先が設定されました。

<OptionalDetails summary="Branchは軽い目印">
  Branchは履歴全体のコピーではなく、Commitを指す移動可能な名前として扱えます。新しいCommitを作ると、現在のBranchだけがそのCommitへ進みます。
</OptionalDetails>

## よくある失敗

Branchを作っただけで安心し、Current Branchを確認せずmainを編集することがあります。編集前・Commit前の2回、現在のBranch名を読みます。

## 対応するGitコマンド

作業フォルダで未Commit変更がないことを確認して実行します。最初のコマンドで作成と切り替え、次でリモートへPublishします。

```bash
git switch -c add-learning-goal
git push -u origin add-learning-goal
```

## 次のレッスンへ

次はこのBranchで変更をCommit・Pushし、mainへ取り込む提案としてPull Requestを作ります。

## なぜリポジトリを作るのか

これから作るプロジェクトのファイルと履歴を、GitHub上で保管する場所が必要です。最初にリモートリポジトリを作ると、次のレッスンで安全に端末へCloneできます。

## 操作前の状態

学習用の作業フォルダ、ローカルリポジトリ、リモートリポジトリはまだありません。

## 手順：GitHubで作成する

1. GitHubにサインインし、画面右上の **＋** から **New repository** を開きます。
2. **Repository name** に `git-guidebook-practice` と入力します。
3. **Description** に「Git Guidebookの練習用リポジトリ」と入力します。
4. 公開してよい練習内容だけなら **Public** を選びます。公開したくない場合は **Private** でも学習できます。
5. **Add a README file** をオンにします。
6. `.gitignore` とLicenseは今回は **None** のままにします。
7. 入力内容を確認し、**Create repository** を選びます。

<Callout type="warning" title="公開範囲を確認">
  Publicは誰でも閲覧できます。個人情報、授業課題の未公開内容、秘密情報は入れないでください。迷う場合はPrivateを選びます。
</Callout>

## 操作後の状態

GitHub上にREADMEを含むリモートリポジトリが作られました。まだ端末内には作業フォルダもローカルリポジトリもありません。

## 内部で起きたこと

GitHubがリモートリポジトリを作り、READMEを追加する最初のCommitも作成しました。READMEは、このリポジトリが何のためにあるかを訪問者へ伝える説明書です。

<OptionalDetails summary=".gitignoreとLicenseはなぜ選ばなかった？">
  `.gitignore`はGitで追跡しないファイルを指定します。Licenseは他の人がコードや文章を利用できる条件を示します。用途ごとの判断が必要なので、この練習では追加しません。特に実プロジェクトのLicenseは所有者が決めます。
</OptionalDetails>

## よくある失敗

同じ名前のリポジトリがすでにあると作成できません。その場合は既存の練習用リポジトリを使うか、`git-guidebook-practice-2`のように名前を変えます。実在する仕事用リポジトリは使わないでください。

## なぜ差分を見るのか

Commitする前に、記録したい変更だけが含まれているかを確かめます。差分を読む習慣があると、秘密情報や関係ない編集を誤って履歴へ入れるリスクを減らせます。

## 操作前の状態

Clone直後なので、作業フォルダ、ローカルリポジトリ、GitHub上の内容は同じです。GitHub Desktopの **Changes** には変更がありません。

## 手順：READMEを編集する

1. 作業フォルダの `README.md` をテキストエディターで開きます。
2. 末尾に次の2行を追加し、ファイルを保存します。

```markdown
## 練習メモ

Gitの3つの場所を区別して学びます。
```

3. GitHub Desktopへ戻り、左側の **Changes** で `README.md` を選びます。
4. 右側の差分で、追加した行だけが表示されているか確認します。
5. ファイル名の左にあるチェックがオンであることを確認します。このチェックは次のCommitに含める変更を選びます。

## 操作後の状態

変わったのは作業フォルダだけです。ローカルリポジトリの最後のCommitとGitHub上の履歴はまだ変わっていません。

## 内部で起きたこと

Gitは作業フォルダの現在内容と、ローカルリポジトリに記録済みの内容を比べ、差分を検出しました。まだ新しいCommitは作られていません。

<OptionalDetails summary="チェックを外すとどうなる？">
  GitHub
  Desktopでファイルのチェックを外すと、その変更は次のCommitに含まれません。作業フォルダのファイルが元に戻るわけではなく、未Commit変更として残ります。
</OptionalDetails>

## よくある失敗

エディターで保存していないため、GitHub Desktopに差分が出ないことがあります。まずファイルを保存し、GitHub Desktopで正しいリポジトリを開いているか確認します。

## 対応するGitコマンド

作業フォルダの中で実行します。`git status` は変更されたファイルを、`git diff` は未Commitの行差分を表示します。どちらも状態を変更しません。

```bash
git status
git diff
```

# スクリーンショット代替ブロック仕様

## 目的

教材ページを実装するとき、必要なスクリーンショットがまだ存在しなくても制作を止めないため、画像を入れたい位置に `ScreenshotPlaceholder` を配置します。

このブロックは仮のイメージ画像ではありません。**後からどのスクリーンショットへ差し替えるかを人間が判断するための、正式な代替表示**です。

教材本文を確認する人は、ブロックの位置と内容を見て次を評価できます。

- 本当に画像が必要か
- 本文に対して配置場所が適切か
- どの画面状態を撮影すべきか
- 1枚で足りるか、操作前後の2枚が必要か
- どの範囲を切り抜くべきか

## 表示要件

代替ブロックは、本文中で見落としにくい色付きの枠として表示します。

ただし、色だけに意味を依存させず、アイコン、見出し、枠線、ラベルを併用します。

表示例：

```text
┌────────────────────────────────────────────┐
│ 📷 スクリーンショット差し替え予定          │
│                                            │
│ ファイル名                                 │
│ lesson-05-step-03-create-commit.png        │
│                                            │
│ 撮影内容                                   │
│ GitHub DesktopでCommitボタンを押す直前     │
│                                            │
│ 必要な状態                                 │
│ README.mdに1行の変更があり、Summary入力済み│
│                                            │
│ 映したい範囲                               │
│ Changes一覧、差分、Summary、Commitボタン   │
└────────────────────────────────────────────┘
```

文言は「画像が壊れています」ではなく、必ず次のどちらかにします。

- `スクリーンショット差し替え予定`
- `スクリーンショット確認待ち`

## 必須情報

各 `ScreenshotPlaceholder` は最低限、次を持ちます。

| 項目 | 内容 |
|---|---|
| `filename` | 最終的に使用する予定の画像名 |
| `title` | 画像で示す操作や状態の短い名前 |
| `description` | 教材上、この画像で何を理解してほしいか |
| `requiredState` | 撮影前に再現するリポジトリやUIの状態 |
| `capture` | 撮影時に行う操作 |
| `crop` | 画像に含めたい画面範囲 |
| `exclude` | 映してはいけない情報 |
| `altDraft` | 完成画像に使用する代替テキストの下書き |

必要に応じて次も指定できます。

| 項目 | 内容 |
|---|---|
| `variant` | `before`、`after`、`dialog`、`error`、`overview`など |
| `priority` | `required`または`optional` |
| `note` | 撮影者や加工担当者への補足 |

## MDXでの使用イメージ

```mdx
<ScreenshotPlaceholder
  filename="lesson-05-step-03-create-commit.png"
  title="Commit直前のChanges画面"
  description="変更内容とCommit対象が対応していることを確認するための画像"
  requiredState={[
    "リポジトリ: git-guidebook-sample",
    "現在のブランチ: main",
    "README.mdに1行の未コミット変更がある",
    "Summaryに『Add first message』と入力済み",
  ]}
  capture="Commit to mainを押す直前で撮影する"
  crop="Changes一覧、差分、Summary入力欄、Commitボタンが読める範囲"
  exclude="メールアドレス、他のリポジトリ名、OSユーザー名"
  altDraft="GitHub DesktopのChanges画面。README.mdの差分とコミットメッセージが表示され、Commit to mainボタンを押す直前の状態"
  priority="required"
/>
```

実装時の記法は変更して構いませんが、表示される情報と役割は維持します。

## 配置ルール

- スクリーンショットを最終的に入れたい**正確な位置**へ配置する
- 章末にまとめて置かない
- 操作説明の前後関係が分かる位置に置く
- 同じ画像を複数箇所で要求しない
- 装飾目的だけの画像には使用しない
- 画像がなくても本文だけで操作できるようにする
- 画像が必要か迷う場合も、一度ブロックを置いて人間が削除判断できるようにする

## `docs/SCREENSHOT_REQUESTS.md`との同期

教材内に代替ブロックを追加したら、同じ内容を `docs/SCREENSHOT_REQUESTS.md` にも登録します。

両者は `filename` を一意な識別子として対応させます。

一覧には最低限、次を記録します。

```markdown
## lesson-05-step-03-create-commit.png

- Lesson: 5 Commit
- 状態: 未撮影
- 優先度: 必須
- 撮影内容: GitHub DesktopでCommitボタンを押す直前
- 掲載位置: 「Commitを実行する」の手順3直後
```

## 人間による確認後の流れ

1. 教材ページを読み、代替ブロックの必要性と位置を確認する
2. 不要ならブロックと依頼一覧を削除する
3. 必要なら指定内容に合わせてスクリーンショットを撮影する
4. 個人情報を確認した生データを `assets/screenshots/raw/` に追加する
5. 切り抜き、注釈、圧縮を行う
6. 完成画像を `public/images/screenshots/` に追加する
7. 代替ブロックを実画像コンポーネントへ差し替える
8. `altDraft`を実画像に合わせて修正する
9. 依頼一覧の状態を完了へ変更する

## 完成画像への差し替え条件

次を満たすまで代替ブロックを削除してはいけません。

- 指定した操作と状態が画像に含まれている
- 個人情報と機密情報が含まれていない
- 本文と現在のUIが一致している
- 必要な文字が読み取れる
- 掲載用ファイル名が一致している
- 適切な代替テキストが確定している
- モバイル表示で画像がページ幅を破壊しない

## 公開時の扱い

MVP制作中は代替ブロックが残っていても構いません。

ただし、ブロックは開発者向けの意味不明なTODO表示にせず、閲覧者にも「この位置には操作画面が追加される予定」と分かる日本語表示にします。

最終公開前には、残っている代替ブロックを一覧化し、必須画像と任意画像を区別して判断します。

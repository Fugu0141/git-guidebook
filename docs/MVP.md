# MVP定義

## 1. MVPの目的

Gitを一度も使ったことがない学習者が、GitHub Desktopを使って一人開発の基本的な流れを体験し、各操作が「どこに」「何を」反映するものなのか説明できる状態を作ります。

MVPでは機能を広げすぎず、最初の成功体験を重視します。

## 2. 対象読者

- Git・GitHubを初めて使う人
- コマンド中心の教材で挫折した人
- Commit、Push、Pullの違いが曖昧な人
- 共同開発に進む前に基本の流れを理解したい人

### 前提知識

- ファイルやフォルダを作成・編集できる
- Webブラウザを操作できる
- GitHubアカウントを作成できる

プログラミング経験は必須にしません。

## 3. 学習ゴール

学習者はMVP修了時に、次のことができる状態を目指します。

- Git、GitHub、GitHub Desktopの役割を自分の言葉で区別できる
- ローカルとリモートの違いを説明できる
- リポジトリを作成し、端末へCloneできる
- 変更差分を確認し、意味のあるメッセージでCommitできる
- CommitとPushの違いを説明できる
- Branchを作り、mainと分離して作業できる
- Pull Requestを作成し、変更内容を確認できる
- Merge後にPullしてローカルを最新にできる
- 操作中に現在地を見失ったとき、状態図から確認できる

## 4. MVPに含める内容

### Lesson 0: なぜバージョン管理が必要か

- `完成版_最終_修正版`問題
- 変更履歴を残す意味
- このコースで完成させるもの

### Lesson 1: Gitの登場人物

- Git
- GitHub
- GitHub Desktop
- ローカルリポジトリ
- リモートリポジトリ

### Lesson 2: リポジトリを作る

- GitHubでリポジトリを作成
- READMEの役割
- Public / Private
- `.gitignore`とLicenseは概要のみ

### Lesson 3: Cloneする

- GitHub上のデータを端末へ持ってくる
- 保存場所の確認
- Clone後の状態図

### Lesson 4: 変更してCommitする

- ファイルを作成・編集
- 差分の見方
- Commitメッセージ
- Commitはローカルへの記録であること

### Lesson 5: Pushする

- Push前後の状態
- GitHub上で反映を確認
- CommitとPushの違い

### Lesson 6: Branchで安全に作業する

- mainの役割
- Branchを分ける理由
- Branchの作成とPublish

### Lesson 7: Pull Requestを作る

- 変更をCommit・Push
- baseとcompare
- Pull Requestのタイトルと説明
- 差分の確認

### Lesson 8: Mergeして取り込む

- Merge前の確認
- Merge
- Branch削除の意味
- GitHub上のmainが更新された状態

### Lesson 9: Pullして同期する

- GitHub上と端末内の差
- FetchとPullの最小限の説明
- ローカルmainを最新にする

### Lesson 10: 振り返り

- 一連の流れを状態図で再確認
- GUI操作とGitコマンドの対応表
- 自力で同じ流れを再現する最終課題

## 5. MVPに含めない内容

次の内容は重要ですが、初期MVPからは除外します。

- CLIのみでの詳細な操作
- Forkを使ったOSS開発
- 複数人によるコードレビュー演習
- Conflictの本格的な解決
- Rebase、Cherry-pick、Resetなどの高度な履歴操作
- Git internalsの詳細
- Git LFS
- GitHub Actionsの利用方法
- 組織向けのBranch protection運用
- 学習アカウント、クラウド同期、ランキング

## 6. 必要なサイト機能

### 必須

- レスポンシブなトップページ
- レッスン一覧
- 前後のレッスンへ移動できるナビゲーション
- 現在地が分かる目次
- Markdownベースの教材ページ
- 画像と図解を見やすく表示できること
- GitHub Pagesへの自動デプロイ
- 404ページ

### 可能ならMVPへ含める

- 学習済みチェックをブラウザに保存
- 用語の補足表示
- 操作前後を切り替える状態図
- 簡単な確認クイズ

## 7. 完了条件

MVPは、次の条件をすべて満たしたときに完成とします。

### コンテンツ

- 全11レッスンに本文がある
- 各操作に操作前・操作後の状態説明がある
- 各レッスンに少なくとも1つ確認項目がある
- Commit、Push、Pull、Branch、Pull Request、Mergeを混同しない説明になっている

### プロダクト

- GitHub Pages上で公開されている
- スマートフォンとPCで本文を読める
- リンク切れがない
- ページの再読み込みや直接URLアクセスで404にならない
- 基本的なキーボード操作ができる

### 検証

- Git未経験者または初心者2人以上に試してもらう
- 少なくとも1人が説明者の補助なしで最終課題まで進める
- 詰まった場所を記録し、重大な問題を修正する

## 8. MVPの成功指標

初期段階ではアクセス数より、学習が成立したかを重視します。

- 完走者が一連の操作順を説明できる
- CommitとPushの違いを説明できる
- BranchからPull Request、Mergeまで再現できる
- 「次に何をすればよいか分からない」状態が教材上の案内で解消できる

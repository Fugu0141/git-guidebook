# スクリーンショット制作ガイド

このドキュメントは、Git Guidebookで使用するスクリーンショットの撮影、確認、加工、配置方法を定めます。

## 基本方針

スクリーンショットは単なる装飾ではなく、学習者が「どこを見て、何を操作するか」を理解するための教材です。

そのため、次の条件を満たす画像だけを使用します。

- 操作対象が明確である
- 本文と同じ手順・状態を示している
- 不要なUIや通知が少ない
- 個人情報や機密情報を含まない
- 画像がなくても本文だけで手順を追える
- 画面読み上げを考慮した代替テキストを付けられる

## 撮影環境

原則として、教材専用の環境を使います。

- 教材専用の公開サンプルリポジトリ
- 架空のファイル名、コミットメッセージ、ユーザー情報
- 通知を停止した状態
- ブラウザの不要なタブやブックマークを閉じた状態
- 必要に応じてOSのユーザー名が映らない保存場所

GitHubやGitHub DesktopのUIは更新されるため、撮影時にはアプリ・OS・ブラウザのおおよそのバージョンをPull Requestへ記録してください。

## 撮影前チェック

- [ ] メールアドレスが映っていない
- [ ] 本名、学校名、住所、電話番号が映っていない
- [ ] Token、API Key、パスワード、QRコードが映っていない
- [ ] 非公開リポジトリや非公開Organizationが映っていない
- [ ] DM、通知、他人のアイコンや氏名が映っていない
- [ ] ローカルPCのユーザー名を含むパスが映っていない
- [ ] ブラウザの他タブやブックマークに個人情報がない
- [ ] 教材で使用する架空データだけが表示されている

一つでも不安がある場合は、その画像をコミットせず撮り直します。ぼかしで隠すより、最初から安全な画面を作って撮影する方を優先します。

## 推奨撮影条件

- PNGを基本とする
- OS標準倍率または100〜125%程度を基準にする
- 文字が読める解像度を確保する
- ウィンドウ全体ではなく、必要な文脈が残る範囲を撮る
- 同じ章ではウィンドウサイズと表示倍率をなるべく統一する
- マウスポインターは必要な場合だけ含める
- ダークモードとライトモードを混在させない

厳密なピクセルサイズは、サイト実装とデザイン決定後に固定します。

## フォルダ構成

```text
assets/
└── screenshots/
    └── raw/          # 未加工の生データ

public/
└── images/
    └── screenshots/  # サイトで使用する完成画像
```

生データを残すことで、UIの切り抜きや注釈方針を後から変更できます。ただし、公開リポジトリであるため、生データにも個人情報を含めてはいけません。

## ファイル名

```text
lesson-<章番号>-step-<手順番号>-<内容>.png
```

例：

```text
lesson-01-step-02-clone-repository.png
lesson-03-step-04-push-origin.png
lesson-06-step-03-create-pull-request.png
```

同じ操作に複数画像が必要な場合：

```text
lesson-03-step-04-push-origin-before.png
lesson-03-step-04-push-origin-after.png
```

## 注釈

矢印、枠、番号などの注釈は必要最小限にします。

- 注釈だけで意味を伝えない
- 色だけで区別しない
- 重要なボタンを過剰に覆わない
- 本文の手順番号と画像内の番号を対応させる
- UIそのものと誤認される装飾を避ける

注釈スタイルはサイトのデザイン決定後に共通化します。

## 代替テキスト

`alt`には画像の見た目ではなく、教材上の役割を書きます。

悪い例：

```text
GitHub Desktopの画像
```

良い例：

```text
GitHub DesktopのChanges画面。変更されたREADME.mdが選択され、左下にコミットメッセージ入力欄が表示されている
```

画像の直前・直後の本文と同じ内容を不必要に繰り返さないようにします。

## 画像を依頼するときの形式

撮影を依頼する際は、Issueまたはタスクリストに次の情報を書きます。

```markdown
### 撮影対象
GitHub Desktopで新しいブランチを作成する画面

### 必要な状態
- リポジトリ: git-guidebook-sample
- 現在のブランチ: main
- 未コミットの変更: なし

### 操作
Current Branch → New Branchを開く

### 映したい範囲
GitHub Desktopの上部とNew Branchダイアログ

### 映してはいけないもの
メールアドレス、他のリポジトリ名、OSユーザー名

### 保存名
lesson-04-step-02-create-branch.png
```

この形式により、撮影者が同じ状態を再現しやすくなります。

## 受け入れチェック

- [ ] 指定された操作と状態が映っている
- [ ] 教材本文とUI表示が一致している
- [ ] 個人情報と機密情報がない
- [ ] 文字が読み取れる
- [ ] 不要な余白や別アプリが映っていない
- [ ] ファイル名が規則に従っている
- [ ] 完成画像に適切な代替テキストを設定できる
- [ ] 使用したソフトウェアのおおよそのバージョンがPRに記録されている

## UI更新への対応

GitHub DesktopやGitHubの画面が変わった場合は、画像だけでなく本文の操作名も確認します。

古い画像を見つけた場合は、次の情報をIssueに記録します。

- 対象レッスン
- 古くなった画像
- 現在のUIとの差
- 撮り直しが必要な範囲

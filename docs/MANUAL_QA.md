# MVP手動確認ガイド

自動検証で確認できない見た目、操作感、教材理解を公開前に確認するための手順です。

## 表示と操作

- 320px、768px、1280px程度の幅でホーム、一覧、Lesson 0、Lesson 5、Lesson 11を確認する
- 横スクロールが発生せず、本文、表、状態図、代替ブロックが読める
- Tabキーだけでスキップリンク、ヘッダー、確認問題、完了記録、前後ナビゲーションへ移動できる
- フォーカス位置が常に見える
- OSの「視差効果を減らす」設定で不要な動きがない
- 確認問題で未選択、不正解、正解のフィードバックが読み上げ可能な位置へ表示される
- 完了記録が再読み込み後も同じブラウザで残る
- ブラウザストレージを無効にしても本文とナビゲーションを利用できる
- 印刷プレビューで本文と状態図が読める

## 教材レビュー

- 初心者2人以上にLesson 0から11まで試してもらう
- 少なくとも1人は説明者の補助なしで最終課題まで進む
- 各参加者について、詰まったLesson、画面、発言、所要時間、必要だった補助を記録する
- CommitとPush、Pull RequestとMerge、FetchとPullの違いを自分の言葉で説明してもらう
- 重大な誤解や完走を妨げる問題を修正し、再確認する

## Lighthouse相当の確認

GitHub Pagesの公開URLで、Chrome DevToolsのLighthouseをモバイル設定で実行します。

- Accessibility、Best Practices、SEOに重大な指摘がない
- 見出し順、ランドマーク、リンク名、フォームラベルを個別に確認する
- 外部Webフォントが取得できない場合も、システムフォントで本文を読める
- 直接URLで各Lessonを開き、再読み込みしても表示できる
- 存在しないURLで404ページが表示される

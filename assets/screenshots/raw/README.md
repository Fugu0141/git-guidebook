# Raw screenshots

このフォルダには、教材用スクリーンショットの**未加工の生データ**を保存します。

サイトから直接参照する画像は、ここではなく `public/images/screenshots/` に配置します。

## 保存するもの

- GitHub Desktopの操作画面
- GitHub Web UIの操作画面
- OSのファイル操作画面
- エラーや確認ダイアログの再現画面
- 加工前の高解像度画像

## 保存しないもの

次の情報が含まれる画像はコミットしないでください。

- メールアドレス、電話番号、住所、学校名などの個人情報
- Access Token、API Key、パスワード、Cookie、QRコード
- 非公開リポジトリ名や社内・学校内プロジェクトの情報
- 他人の氏名、アイコン、通知内容、DMなど
- ローカルPCのユーザー名を含むパス
- 公開する権利のない画像や文章

公開GitHubユーザー名や公開リポジトリ名を映す場合も、教材上必要かを確認してください。原則として、教材専用のサンプルリポジトリと架空データを使います。

## ファイル名

次の形式を基本とします。

```text
lesson-<章番号>-step-<手順番号>-<内容>.<拡張子>
```

例：

```text
lesson-02-step-03-create-commit.png
lesson-04-step-02-create-branch.png
lesson-06-step-05-open-pull-request.png
```

- 半角英数字とハイフンを使う
- 内容は短い英語で表す
- `final`、`new`、`修正版2`のような版管理をファイル名で行わない
- 差し替え履歴はGitで管理する

## 撮影後の流れ

1. 個人情報と機密情報を確認する
2. このフォルダへ未加工画像を保存する
3. 必要に応じて切り抜き、注釈、圧縮を行う
4. 完成画像を `public/images/screenshots/` に保存する
5. 教材ページから完成画像だけを参照する

詳しい撮影・確認手順は [`docs/SCREENSHOT_GUIDE.md`](../../../docs/SCREENSHOT_GUIDE.md) を参照してください。
